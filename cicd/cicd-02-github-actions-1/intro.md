# GitHub Actions: Часть 1

В предыдущем модуле мы построили CI/CD пайплайн **вручную на bash**. Мы поняли механику: запустили команды по очереди, при ошибке — остановились. Теперь делаем то же самое, но **профессиональным инструментом**.

**GitHub Actions** — встроенная в GitHub система CI/CD. Вместо того чтобы писать bash-скрипт, вы описываете пайплайн в YAML-файле, кладёте его в репозиторий — и GitHub автоматически запускает его при каждом `git push`.

## Почему GitHub Actions популярен?

- **Встроен в GitHub** — не нужно настраивать отдельный сервер
- **Бесплатно** для публичных репозиториев, 2000 минут/месяц для приватных
- **Огромный маркетплейс** — тысячи готовых Actions для любых задач
- **YAML-конфиг в репозитории** — версионируется вместе с кодом

## Как мы будем практиковаться без интернета?

На Killercoda нет доступа к GitHub.com, но есть инструмент **`act`** — он запускает GitHub Actions workflow **локально**, используя Docker. Это не имитация — это тот же самый синтаксис, те же самые файлы, только выполняется на вашей машине.

> Проверьте Docker: `docker --version`{{execute}}

> Устанавливаем `act`:
```bash
cd /tmp && wget -qO act.tar.gz https://github.com/nektos/act/releases/latest/download/act_Linux_x86_64.tar.gz && tar xzf act.tar.gz && mv act /usr/local/bin/ && chmod +x /usr/local/bin/act && act --version
```{{execute}}
