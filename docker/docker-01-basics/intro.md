## Урок 1: Первые шаги с Docker

До Docker проблема звучала так: "На моей машине работает, а на сервере — нет."
Причина — разные версии библиотек, ОС, конфигураций.

Docker решает это упаковывая приложение вместе со всем окружением в **контейнер**.
Контейнер запускается одинаково везде: на ноутбуке разработчика, на CI/CD и на продакшн-сервере.

---

### Главные понятия:

**Image (образ)** — "замороженный" шаблон файловой системы. Неизменяемый.
**Container (контейнер)** — запущенный экземпляр образа. Изолированный процесс.
**Docker Hub** — публичный реестр образов, как npm для node или pip для python.
