## Как Docker работает изнутри

Прежде чем запускать команды, важно понять что происходит под капотом.

Когда вы пишете `docker run nginx`:
1. Docker проверяет: есть ли образ `nginx` локально?
2. Если нет — скачивает его с Docker Hub
3. Создаёт контейнер (изолированный процесс) на основе образа
4. Запускает команду по умолчанию из образа

---

**Задание:**

1. Проверьте что Docker установлен и работает:
`docker --version`

2. Посмотрите статус Docker-демона:
`docker info`

3. Сохраните версию в файл:
`docker --version > /root/docker_version.txt`

4. Убедитесь, что файл создался:
`cat /root/docker_version.txt`
