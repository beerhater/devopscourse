# Шаг 4: Остановка контейнера — docker stop

`docker stop` **плавно останавливает** запущенный контейнер:
1. Отправляет сигнал `SIGTERM` процессу внутри контейнера
2. Ждёт 10 секунд (по умолчанию)
3. Если контейнер не остановился — принудительно завершает через `SIGKILL`

## Задание 1: Остановка по имени

```bash
docker stop my-nginx
```{{execute}}

## Задание 2: Проверка статуса

```bash
docker ps -a --filter "name=my-nginx"
```{{execute}}

Статус изменится с `Up` на `Exited`.

## Задание 3: Остановка с таймаутом

Флаг `-t` задаёт время ожидания в секундах:

```bash
docker run -d --name test-stop nginx
docker stop -t 2 test-stop
```{{execute}}

> **Отличие от `docker kill`:** `docker stop` даёт процессу время завершиться корректно, `docker kill` убивает немедленно.
