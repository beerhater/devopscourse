## Заходим внутрь контейнера

Иногда нужно "зайти" внутрь контейнера для отладки.
Команда `docker exec` выполняет команду **внутри работающего контейнера**.

Флаги `-it`:
- `-i` (interactive) — интерактивный режим (передаём stdin)
- `-t` (tty) — выделяем псевдотерминал

Вместе они дают вам шелл внутри контейнера.

---

1. Зайдите внутрь контейнера `web`:
`docker exec -it web bash`

2. Теперь вы **внутри контейнера**. Убедитесь в этом:
`hostname`
`cat /etc/os-release`

3. Посмотрите файлы nginx:
`ls /usr/share/nginx/html/`

4. Выйдите из контейнера:
`exit`

5. Выполните команду внутри контейнера не заходя в него:
`docker exec web cat /etc/nginx/nginx.conf`
