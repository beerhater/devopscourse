## Остановка и удаление контейнеров

Умение чисто останавливать и удалять контейнеры — обязательный навык.

`docker stop` отправляет сигнал SIGTERM (вежливая просьба завершиться),
ждёт 10 секунд и затем SIGKILL.
`docker kill` — немедленный SIGKILL без ожидания.

---

1. Остановите контейнер `web`:
`docker stop web`

2. Проверьте — он должен быть в статусе `Exited`:
`docker ps -a`

3. Удалите его:
`docker rm web`

4. Удалите контейнер hello-world который остался с шага 2:
`docker rm $(docker ps -a -q)`

5. Убедитесь что список пуст:
`docker ps -a`

> **Совет:** `docker run --rm` автоматически удаляет контейнер после завершения.
> Удобно для разовых задач.
