# Шаг 1: Что такое образ Docker

**Образ (image)** — это неизменяемый шаблон, из которого создаются контейнеры. Думайте об образе как о рецепте: из одного рецепта можно приготовить много блюд (контейнеров).

Образ состоит из **слоёв (layers)**:
- Каждая инструкция в Dockerfile создаёт новый слой
- Слои кешируются и переиспользуются — это экономит место и время

## Откуда берутся образы?

Образы хранятся в **реестрах (registry)**. Главный публичный реестр — **Docker Hub** (hub.docker.com). Там тысячи готовых образов: `nginx`, `ubuntu`, `postgres`, `redis` и другие.

## Посмотрите, какие образы уже есть локально

```bash
docker images
```{{execute}}

Скорее всего, список пустой или содержит образы из предыдущего модуля. Давайте это исправим!

> **Формат имени образа:** `[registry/][namespace/]name[:tag]`
> Например: `docker.io/library/nginx:latest` → сокращённо просто `nginx`
