# Шаг 4: Теги образов и версионирование

**Тег (tag)** — это метка версии образа. Один образ может иметь несколько тегов.

## Тег latest

Если тег не указан, Docker использует `latest` — это не обязательно самая новая версия, просто тег по умолчанию:

```bash
docker pull ubuntu
# то же самое что:
docker pull ubuntu:latest
```{{execute}}

## Конкретные версии

Всегда лучше указывать конкретный тег в продакшене — `latest` может сломать ваш код при обновлении:

```bash
docker pull ubuntu:22.04
```{{execute}}

```bash
docker pull ubuntu:20.04
```{{execute}}

## Посмотрите все скачанные версии ubuntu

```bash
docker images ubuntu
```{{execute}}

Вы увидите несколько строк с разными тегами — `latest`, `22.04`, `20.04`. Это разные образы, но с одним именем.

## Задание

Скачайте `nginx` версии `1.25`:

```bash
docker pull nginx:1.25
```{{execute}}

Сравните размеры разных версий:

```bash
docker images nginx
```{{execute}}
