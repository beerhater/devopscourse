## Что вы освоили

- **`FROM`** — базовый образ, важность конкретных тегов
- **`RUN`** — объединение команд через `&&` для уменьшения слоёв
- **`COPY`** — копирование файлов, отличие от `ADD`
- **`CMD`** — команда по умолчанию, переопределяемая при `docker run`
- **`ENTRYPOINT`** — фиксированная точка входа
- **`WORKDIR`** — рабочая директория
- **`ENV`** — переменные окружения для сборки и runtime
- **`EXPOSE`** — документирование портов
- **`ARG`** — аргументы сборки через `--build-arg`
- **`.dockerignore`** — исключение лишних файлов

## Шпаргалка

| Инструкция | Назначение |
|-----------|-----------|
| `FROM image:tag` | Базовый образ |
| `RUN cmd1 && cmd2` | Команда при сборке |
| `COPY src dest` | Копировать файлы |
| `WORKDIR /path` | Рабочая директория |
| `ENV KEY=VALUE` | Переменная окружения |
| `ARG NAME=default` | Аргумент сборки |
| `EXPOSE 8080` | Документировать порт |
| `CMD ["exec", "arg"]` | Команда по умолчанию |
| `ENTRYPOINT ["exec"]` | Фиксированная точка входа |
