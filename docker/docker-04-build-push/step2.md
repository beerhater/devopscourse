## Кэш слоёв при сборке

Docker кэширует каждый слой. Если слой не изменился — он берётся из кэша
и не пересобирается. Это делает повторные сборки очень быстрыми.

Строка `---> Using cache` означает: этот слой взят из кэша.

---

1. Соберите образ ещё раз — заметьте как он собирается мгновенно:
`docker build -t webserver:v1 .`

2. Теперь измените index.html и пересоберите:
`echo "<h1>Updated!</h1>" > /root/webserver/index.html`

`docker build -t webserver:v2 .`

Первый слой (FROM nginx:alpine) — взят из кэша.
Второй слой (COPY) — пересобран, так как файл изменился.

3. Чтобы сбросить весь кэш: `docker build --no-cache -t webserver:fresh .`
