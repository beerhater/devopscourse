# Docker: Тома и данные

Контейнеры **эфемерны** — при удалении контейнера все данные внутри него исчезают. Это отлично для stateless-приложений, но что если нам нужна база данных или загруженные пользователем файлы?

Docker решает эту проблему тремя способами хранения данных:

- **Именованные тома (volumes)** — управляются Docker, рекомендованный способ
- **Bind mounts** — монтирование директории с хоста в контейнер
- **tmpfs** — данные хранятся только в памяти (не на диске)

В этом модуле вы освоите каждый из подходов и поймёте, когда какой использовать.
