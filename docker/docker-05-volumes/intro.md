## Урок 5: Тома и данные

Контейнеры — временные. Когда вы делаете `docker rm`, все данные внутри пропадают.

Это хорошо для приложений (stateless), но плохо для баз данных и файлового хранилища.

---

Docker предлагает три способа хранить данные:

| Тип         | Где хранится                 | Когда использовать              |
|-------------|------------------------------|---------------------------------|
| **Volume**  | Docker управляет сам         | Базы данных, постоянные данные  |
| **Bind**    | Любая папка на хосте         | Разработка, конфиги             |
| **tmpfs**   | Только в памяти              | Временные секреты               |

---

Самый правильный выбор для продакшна — **Named Volume**.
Docker сам управляет где и как хранятся данные.
