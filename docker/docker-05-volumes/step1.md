# Шаг 1: Проблема персистентности данных

Убедимся на практике, что данные в контейнере исчезают при его удалении.

## Демонстрация проблемы

Создайте контейнер и запишите в него данные:

```bash
docker run --name data-test alpine sh -c "echo 'Important data' > /data/file.txt && cat /data/file.txt"
```{{execute}}

Удалите контейнер:

```bash
docker rm data-test
```{{execute}}

Запустите новый контейнер из того же образа:

```bash
docker run --rm alpine cat /data/file.txt
```{{execute}}

Файла нет! Данные потеряны вместе с контейнером.

## Почему так происходит?

Каждый контейнер получает свой **writable layer** поверх слоёв образа. Все изменения (новые файлы, измененные файлы) пишутся в этот слой. При удалении контейнера — writable layer удаляется вместе с ним.

```
[образ alpine: read-only слои]
        ↓
[writable layer контейнера] ← ваши данные здесь, исчезнут при docker rm
```

Решение — вынести данные **за пределы** writable layer с помощью томов.
