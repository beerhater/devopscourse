# Поздравляем! 🎉

Вы успешно завершили модуль **«Docker: Сеть»**!

## Что вы освоили

- **Сетевые драйверы** — `bridge`, `host`, `none` и когда каждый применять
- **Дефолтная bridge** — связь по IP, но без DNS
- **Пользовательские сети** — `docker network create`, встроенный DNS
- **DNS между контейнерами** — обращение по имени и псевдонимам
- **Несколько сетей** — `docker network connect/disconnect`
- **Проброс портов** — `-p host:container`, `127.0.0.1` binding
- **Управление** — `inspect`, `rm`, `prune`

## Сравнение драйверов

| Драйвер | Изоляция | DNS | Производительность | Применение |
|---------|----------|-----|-------------------|------------|
| bridge (default) | Да | Нет | Средняя | Простые сценарии |
| bridge (custom) | Да | **Да** | Средняя | **Рекомендуется** |
| host | Нет | — | Максимальная | Высоконагруженные сервисы |
| none | Полная | — | — | Batch, безопасность |

## Шпаргалка

| Команда | Описание |
|---------|----------|
| `docker network ls` | Список сетей |
| `docker network create NAME` | Создать сеть |
| `docker network inspect NAME` | Подробности |
| `docker network connect NET CONT` | Подключить контейнер |
| `docker network disconnect NET CONT` | Отключить |
| `docker network rm NAME` | Удалить сеть |
| `docker network prune` | Удалить неиспользуемые |
| `-p 8080:80` | Пробросить порт |
| `--network NAME` | Указать сеть при запуске |
