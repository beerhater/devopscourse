## Урок 6: Сеть в Docker

Когда вы запускаете несколько контейнеров — им нужно общаться друг с другом.
Например: контейнер с приложением должен достучаться до контейнера с базой данных.

---

### Типы сетей Docker:

| Драйвер    | Что это                                                   |
|------------|-----------------------------------------------------------|
| `bridge`   | Виртуальная сеть на хосте. Стандарт по умолчанию.        |
| `host`     | Контейнер использует сеть хоста напрямую                  |
| `none`     | Полная изоляция. Нет сети вообще.                         |
| `overlay`  | Для нескольких Docker-хостов (Docker Swarm/Kubernetes)    |

---

Для большинства задач используется `bridge`.
Главное правило: **контейнеры в одной пользовательской сети находят
друг друга по имени контейнера** — без IP-адресов.
