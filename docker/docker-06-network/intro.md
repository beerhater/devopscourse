# Docker: Сеть

Реальные приложения редко работают в одном контейнере. Веб-сервер должен общаться с базой данных, кеш — с бекендом. Docker предоставляет гибкую сетевую модель для связи контейнеров.

В этом модуле вы освоите:

- **Сетевые драйверы** — `bridge`, `host`, `none`
- **`docker network create`** — пользовательские сети
- **DNS между контейнерами** — обращение по имени контейнера
- **`docker network connect/disconnect`** — подключение к нескольким сетям
- **Проброс портов** — публикация сервисов наружу

> Проверьте среду: `docker network ls`{{execute}}
