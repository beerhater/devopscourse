# Шаг 1: Сетевые драйверы Docker

Docker поддерживает несколько **сетевых драйверов** — каждый для своей задачи.

## Посмотрите существующие сети

```bash
docker network ls
```{{execute}}

Docker создаёт три сети по умолчанию:

| Сеть | Драйвер | Описание |
|------|---------|----------|
| `bridge` | bridge | Изолированная сеть, контейнеры общаются через IP |
| `host` | host | Контейнер использует сеть хоста напрямую |
| `none` | null | Нет сети вообще — полная изоляция |

## Основные драйверы

**bridge** — драйвер по умолчанию. Создаёт виртуальный коммутатор (`docker0`) на хосте. Контейнеры в одной bridge-сети видят друг друга.

**host** — контейнер разделяет сетевой стек хоста. Нет изоляции, но максимальная производительность.

**overlay** — для Docker Swarm, связывает контейнеры на разных хостах.

**none** — контейнер без сети. Только loopback (`lo`).

## Подробности сети bridge

```bash
docker network inspect bridge
```{{execute}}

Обратите внимание на `Subnet` — диапазон IP-адресов для контейнеров в этой сети.
