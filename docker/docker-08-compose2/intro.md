# Docker Compose: Часть 2

В первой части вы освоили базовый запуск стека. Теперь — продвинутые возможности для реальных проектов.

В этом модуле вы освоите:

- **`depends_on` с `condition`** — ждать готовности сервиса, а не просто его запуска
- **`healthcheck`** — проверка что сервис действительно работает
- **`build`** — сборка образов прямо из Compose
- **`--scale`** — запуск нескольких реплик сервиса
- **`profiles`** — опциональные сервисы для dev/tools
- **Override-файлы** — разные конфиги для dev/prod без дублирования

> Проверьте: `docker-compose version`{{execute}}
