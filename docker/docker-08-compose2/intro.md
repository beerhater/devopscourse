## Урок 8: Docker Compose (Часть 2)

Базовый Compose вы уже знаете. В этом уроке добавим профессиональные практики:

**`depends_on`** — управление порядком запуска сервисов.

**`.env` файл** — вынести секреты (пароли, ключи) из compose-файла.
Это позволяет хранить `docker-compose.yml` в git, не раскрывая секреты.

**`healthcheck`** — проверка готовности сервиса.
База данных "запущена" не значит "готова принимать соединения".

**`scale`** — запустить несколько реплик одного сервиса.
