# Шаг 5: Проверяем проброс порта

Контейнер может быть здоров, но наружу сервис всё равно не выйдет, если порт проброшен не туда.

```bash
docker port inspect-demo 80 > /root/inspect_ports.txt
cat /root/inspect_ports.txt
```{{execute}}

Это быстрый способ подтвердить, что `80` внутри контейнера действительно связан с `8091` на хосте.
