# Шаг 2: Control Plane (Мозг кластера)

Control Plane принимает все решения. В нашем кластере он находится на узле `controlplane`.

Посмотрим на компоненты мозга. В K8s системные процессы запускаются тоже в виде контейнеров (подов) в специальном системном пространстве имён `kube-system`:

```bash
kubectl get pods -n kube-system
```{{execute}}

Что есть что:
1. **kube-apiserver** — ЕДИНСТВЕННАЯ точка входа. Все остальные компоненты и пользователи общаются только с ним.
2. **etcd** — сверхнадёжная Key-Value база данных. Хранит вообще всё: настройки, состояния, секреты. Если etcd безвозвратно умрёт — кластер умрёт.
3. **kube-scheduler** — когда появляется новый Под, планировщик смотрит на ресурсы узлов (CPU, память) и решает, на какой узел его посадить.
4. **kube-controller-manager** — бесконечный цикл, который постоянно сверяет *текущее* состояние с *желаемым*. (Например: "должно работать 3 пода, а работает 2 → командую запустить ещё один").
