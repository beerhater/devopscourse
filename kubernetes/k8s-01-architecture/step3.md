# Шаг 3: Worker Node (Рабочие узлы)

Worker Node — это сервер, на котором реально работают ваши приложения. В нашем окружении это узел `node01`.

Посмотрим детальную информацию об узлах:
```bash
kubectl get nodes -o wide
```{{execute}}

Обратите внимание на колонку **CONTAINER-RUNTIME**. Там указан `containerd` — современная и более лёгкая замена старому Docker-демону.

## Компоненты Worker Node

1. **Kubelet** — главный агент на сервере. Он слушает команды от API-сервера ("запусти такой-то под") и дёргает Container Runtime, чтобы тот скачал образ и поднял контейнер.
2. **Kube-proxy** — отвечает за сеть. Именно он настраивает правила маршрутизации (`iptables` или `IPVS`) на сервере так, чтобы трафик мог дойти до вашего пода.
3. **Container Runtime** — (containerd, CRI-O). Занимается "грязной работой": распаковывает слои образов, изолирует процессы.

Давайте зайдём по SSH на рабочий узел и посмотрим, что агент Kubelet реально работает как системная служба Linux:
```bash
ssh node01 "systemctl status kubelet --no-pager | head -n 10"
```{{execute}}
