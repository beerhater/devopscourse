# Deployment — самовосстановление и обновления

Pod сам по себе не восстанавливается — удалил и всё. **Deployment** решает эту проблему: он следит за нужным количеством подов и автоматически пересоздаёт упавшие.

В реальных проектах вы **никогда** не создаёте Pod напрямую — только через Deployment.

## Что изучим

- `kubectl create deployment` — создать деплоймент
- `replicas` — сколько копий пода держать живыми
- `kubectl rollout` — история обновлений и откат
- `kubectl scale` — изменить количество реплик
- `kubectl set image` — обновить образ без downtime
- Стратегии обновления: `RollingUpdate` vs `Recreate`
- Полный YAML-манифест Deployment

> Проверим кластер: `kubectl get nodes`{{execute}}
