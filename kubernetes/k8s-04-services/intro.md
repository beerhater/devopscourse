# Service — стабильный сетевой адрес для подов

Поды эфемерны: их IP меняется при каждом перезапуске. **Service** даёт постоянный адрес и балансирует трафик между репликами.

## Типы Service

| Тип | Доступен | Для чего |
|-----|---------|----------|
| `ClusterIP` | Только внутри кластера | Связь между сервисами |
| `NodePort` | С любого узла по порту | Dev/тестирование |
| `LoadBalancer` | Из интернета | Продакшн (cloud) |
| `ExternalName` | DNS алиас | Внешние сервисы |

## Что изучим

- Почему IP пода ненадёжен
- `ClusterIP` — внутренний сервис
- `NodePort` — доступ снаружи через порт узла
- `LoadBalancer` — облачный балансировщик
- `selector` — как Service находит свои поды
- `Endpoints` — что реально стоит за Service
- `kube-dns` — обращение по имени сервиса

> Проверяем кластер: `kubectl get nodes`{{execute}}
