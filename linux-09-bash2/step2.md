## Проверка аргументов

Хороший скрипт всегда проверяет что нужные данные были переданы.
Иначе он упадёт с непонятной ошибкой — или молча сделает что-то не то.

Для этого используется флаг `-z` (zero) — он проверяет **пустая ли переменная**.

```
if [ -z "$1" ]; then
    echo "Ошибка! Аргумент не передан."
    exit 1
fi
```

`exit 1` — это принудительная остановка скрипта с кодом ошибки.
Код `0` = успех, любой другой = ошибка. Это стандарт Unix.

---

**Шаг 1.** Создайте и откройте скрипт деплоя:
`nano deploy.sh`

**Шаг 2.** Введите код:
```
#!/bin/bash
if [ -z "$1" ]; then
    echo "Ошибка! Укажите окружение."
    echo "Использование: ./deploy.sh [staging|production]"
    exit 1
fi
echo "Деплоим в окружение: $1..."
echo "Деплой завершён успешно!"
```
Сохраните: `Ctrl+O` → Enter → `Ctrl+X`

**Шаг 3.** Сделайте исполняемым:
`chmod +x deploy.sh`

**Шаг 4.** Запустите БЕЗ аргумента — должна появиться ошибка:
`./deploy.sh`

**Шаг 5.** Теперь запустите С аргументом:
`./deploy.sh staging`
