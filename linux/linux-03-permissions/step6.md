## Настраиваем umask

`umask` задаёт какие права **запрещены по умолчанию** для новых файлов и директорий.

Это очень практичная тема:
- чтобы новые файлы не создавались слишком открытыми;
- чтобы секреты и конфиги сразу имели безопасные права;
- чтобы не приходилось каждый раз вручную делать `chmod`.

Напоминание:
- новый файл обычно стартует с базы `666`;
- новая директория — с базы `777`;
- затем из этих значений вычитается `umask`.

**Задание:**
1. Установите маску: `umask 027`
2. Создайте новый файл: `touch secure-by-default.txt`
3. Проверьте права: `ls -la secure-by-default.txt`
4. Дополнительно проверьте цифрами: `stat -c "%a" secure-by-default.txt`

Ожидаемый результат для файла: `640`.
Это хороший практический пример: владелец читает и пишет, группа читает, остальные ничего не видят.
