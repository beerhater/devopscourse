## 🎉 Урок 6 пройден!

Теперь вы умеете диагностировать сеть как настоящий DevOps-инженер:

- `ping -c 4 хост` — проверить доступность
- `curl -I url` — проверить HTTP-ответ сервиса
- `curl -s -o /dev/null -w "%{http_code}" url` — получить HTTP-код
- `wget -O файл url` — скачать файл
- `ss -tulnp` — посмотреть открытые порты
- `netstat -tulnp | grep :порт` — найти процесс на порту
- `ip addr` — посмотреть IP-адреса интерфейсов
- `ip route` — посмотреть маршруты
- `iptables -L -n` — посмотреть правила файрвола
- `iptables -A INPUT -p tcp --dport порт -j ACCEPT` — открыть порт

В следующем уроке разберём пакетный менеджер apt —
как устанавливать, обновлять и удалять пакеты.
