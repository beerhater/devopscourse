## Урок 6: Сеть в Linux

Любой DevOps-инженер большую часть времени работает с сетью.
Развернуть сервис мало — нужно убедиться что он доступен,
слушает нужный порт и не блокируется файрволом.

Классическая ситуация: приложение запущено, но не отвечает.
Что делать? Именно для этого нужны инструменты из этого урока.

В этом уроке вы научитесь:
- Проверять доступность хостов через `ping`
- Делать HTTP-запросы через `curl`
- Скачивать файлы через `wget`
- Смотреть открытые порты через `ss` и `netstat`
- Просматривать сетевые интерфейсы через `ip`
- Управлять базовыми правилами файрвола через `iptables`
