## Логирование в скриптах

Cron-задачи запускаются ночью без вашего присутствия.
Если что-то пошло не так — вы должны об этом узнать утром из лог-файла.

Стандартный подход: создать функцию `log()` которая пишет сообщение
одновременно в терминал И в файл, с меткой времени.

`tee` — команда которая дублирует вывод: пишет и в файл и на экран одновременно.
`-a` (append) — добавлять в конец файла, не перезаписывать.

---

**Шаг 1.** Создайте скрипт:
`nano logger.sh`

**Шаг 2.** Введите код:
```
#!/bin/bash
LOGFILE="/root/script.log"

function log() {
    TIMESTAMP=$(date '+%Y-%m-%d %H:%M:%S')
    echo "[$TIMESTAMP] $1" | tee -a "$LOGFILE"
}

log "Скрипт запущен"
log "Проверяем окружение..."
sleep 1
log "Всё в порядке!"
log "Скрипт завершён"
```
Сохраните: `Ctrl+O` → Enter → `Ctrl+X`

**Шаг 3.** Запустите дважды:
`chmod +x logger.sh && ./logger.sh`
`./logger.sh`

**Шаг 4.** Посмотрите лог — все запуски записались:
`cat /root/script.log`
