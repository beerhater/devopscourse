## Фильтрация логов

`journalctl` — мощный инструмент с богатыми возможностями фильтрации.
В реальных ситуациях вам нужно быстро найти ошибку среди тысяч строк.

**Фильтрация по приоритету (`-p`):**
```
emerg   — система неработоспособна
alert   — требуется немедленное действие
crit    — критическая ошибка
err     — ошибка
warning — предупреждение
notice  — важное сообщение
info    — информация
debug   — отладка
```

**Другие полезные флаги:**
```
-b          — логи с последней загрузки
-b -1       — логи с предыдущей загрузки
--since "2026-03-16 10:00:00"  — с определённого времени
--until "2026-03-16 11:00:00"  — до определённого времени
-o json     — вывод в JSON
```

---

**Шаг 1.** Покажите только ошибки всей системы:
`journalctl -p err -b`

**Шаг 2.** Покажите логи nginx только за последний час:
`journalctl -u nginx --since "1 hour ago" > /root/nginx_hour.log`

**Шаг 3.** Посмотрите сколько места занимают все логи на диске:
`journalctl --disk-usage`

**Шаг 4.** Посмотрите логи в формате JSON (первые 3 записи):
`journalctl -u nginx -o json -n 3`
