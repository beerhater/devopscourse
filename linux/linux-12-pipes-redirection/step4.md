## Перенаправляем ошибки через 2>

`2>` означает: "отправь stderr в отдельный файл".

Это полезно, когда нужно:
- сохранить только ошибки;
- не смешивать их с нормальным выводом;
- потом отдельно разобрать, что пошло не так.

```bash
ls /etc/passwd /etc/hosts /opt/pipes-lab/absent.cfg > /root/ls_ok.txt 2> /root/ls_errors.txt

cat /root/ls_ok.txt
cat /root/ls_errors.txt
```{{execute}}

Теперь успешные результаты и ошибки лежат отдельно — это гораздо удобнее для диагностики.
