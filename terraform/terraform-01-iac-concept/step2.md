# Шаг 2: ClickOps vs IaC

## Проблемы ClickOps

```bash
cat << 'EOF'
Представь: нужно поднять одинаковую инфраструктуру в 3 окружениях:
  dev / staging / production

ClickOps:
  ❌ Делаешь всё вручную три раза
  ❌ В staging забыл добавить Security Group
  ❌ Production отличается от staging — непонятно чем именно
  ❌ Уволился коллега, никто не знает как всё настроено
  ❌ Нельзя сделать code review инфраструктуры
  ❌ Откат при ошибке — снова руками

IaC с Terraform:
  ✅ Одни и те же .tf файлы для всех трёх окружений
  ✅ Разница между окружениями — только в переменных
  ✅ Всё в Git: history, blame, code review, pull request
  ✅ Новый коллега читает .tf и понимает всю инфраструктуру
  ✅ Откат: git revert + terraform apply
  ✅ CI/CD: инфраструктура деплоится автоматически
EOF
```{{execute}}

## Декларативный vs Императивный

```bash
cat << 'EOF'
ИМПЕРАТИВНЫЙ подход (КАК делать):
  Bash-скрипт:
    создать VPC
    создать subnet в VPC
    создать security group
    запустить EC2 с этими параметрами
    если EC2 уже есть — пропустить
    если security group изменилась — обновить...

  Проблема: ты управляешь процессом и должен учесть все состояния

ДЕКЛАРАТИВНЫЙ подход (ЧТО должно быть):
  Terraform HCL:
    resource "aws_instance" "web" {
      ami           = "ami-abc123"
      instance_type = "t2.micro"
    }

  Terraform сам разберётся:
    - Нет EC2? → создать
    - EC2 есть с нужными параметрами? → ничего не делать
    - EC2 есть, но тип изменился? → пересоздать
    - Ресурс удалён из .tf? → удалить в облаке
EOF
```{{execute}}

## Другие IaC-инструменты

```bash
cat << 'EOF'
Инструмент      Подход         Область применения
──────────────  ─────────────  ──────────────────────────────────
Terraform       Декларативный  Провижнинг облачной инфраструктуры
Ansible         Процедурный    Конфигурирование ОС и приложений
Pulumi          Императивный   IaC на Python/TypeScript/Go
CloudFormation  Декларативный  Только AWS (YAML/JSON)
Crossplane      Декларативный  Kubernetes-native IaC

Terraform vs Ansible:
  Terraform  <- создать сервер, сеть, базу данных, S3 bucket
  Ansible    <- настроить ОС, установить nginx, задеплоить приложение
  Вместе     <- Terraform создаёт инфраструктуру → Ansible её настраивает
EOF
```{{execute}}
